# Calculator App

A functional desktop calculator built in Python with a Tkinter GUI and three
selectable visual themes: **Classic**, **Dark (Kiro)**, and **Fun**.

## Demo Video

A short demo showing the app working:
[`Kiro-Birthday-Challenge-Day-4.mov`](./Kiro-Birthday-Challenge-Day-4.mov)
(click **View raw** / **Download** on GitHub to play it).

## 1. Description

Calculator App is a standard arithmetic calculator you operate through on-screen
buttons and a live display. It supports:

- **Arithmetic**: addition, subtraction, multiplication, and division, with
  results accurate to a tight tolerance and formatted to 10 significant digits.
- **Numeric input**: multi-digit operands (up to 15 digits), decimal points with
  a single-decimal-per-operand rule, and a leading-zero convenience.
- **Division-by-zero handling**: shows a clear "Cannot divide by zero" message
  instead of crashing, and recovers on the next input.
- **Clear & delete**: reset the whole expression or remove the last character,
  always flooring the display at a single zero.
- **Three themes**: Classic (light default), Dark (Kiro), and Fun (colorful),
  switchable at runtime. Changing a theme is purely visual and never disturbs
  your in-progress calculation.

### Architecture

The app follows a strict inward dependency direction so the logic is fully
testable without a live window:

| Component | File | Role |
|-----------|------|------|
| `Arithmetic_Engine` | `calculator/arithmetic_engine.py` | Pure two-operand evaluation; returns explicit `Ok`/`Err` values |
| `CalculatorState` | `calculator/calculator_state.py` | Pure input/editing state model and display formatting |
| `ThemeManager` | `calculator/theme_manager.py` | Applies the three themes; tracks the single active theme |
| `CalculatorGUI` | `calculator/gui.py` | Thin Tkinter view/controller |
| Entry point | `calculator/__main__.py` | Builds the window and runs the main loop |

## 2. Built with a single prompt using Kiro Specs

This entire application was produced from **one prompt** given to Kiro:

> "Create a functional Calculator app based on Python having GUI with 3 themes
> classic, dark (Kiro) and fun."

Rather than hand-coding from that prompt, the work went through Kiro's
**spec-driven development** workflow, which turned the rough idea into
structured, reviewable artifacts before any code was written:

1. **Requirements** (`.kiro/specs/calculator-app/requirements.md`) — the idea was
   expanded into user stories and precise, testable acceptance criteria in EARS
   format (numeric input, arithmetic, division-by-zero, clear/delete, GUI, and
   theming).
2. **Design** (`.kiro/specs/calculator-app/design.md`) — the technical design
   defined the component architecture, data models, error handling, and a set of
   14 correctness properties for property-based testing.
3. **Tasks** (`.kiro/specs/calculator-app/tasks.md`) — the design was broken into
   an ordered, incremental implementation plan, each task traceable back to
   specific requirements and design properties.

The implementation was then generated by executing those tasks. The spec files
remain in `.kiro/specs/calculator-app/` as living documentation of how the app
was designed and built.

## 3. Running the app from scratch on a new machine

### Prerequisites

- **Python 3.10 or newer** (`python3 --version` to check).
- **Tkinter** — bundled with most Python installations. If it's missing:
  - macOS (Homebrew): `brew install python-tk`
  - Debian/Ubuntu: `sudo apt-get install python3-tk`
  - Windows: included with the official python.org installer.

### Setup

```bash
# 1. Get the project onto the machine, then move into it
cd "Day 4 Challenge"

# 2. Create and activate a virtual environment
python3 -m venv .venv
source .venv/bin/activate        # macOS/Linux
# .venv\Scripts\activate         # Windows (PowerShell/cmd)

# 3. Install dependencies (needed only for running the tests;
#    the app itself uses just the standard library)
pip install -r requirements.txt
```

### Launch

```bash
python -m calculator
```

The calculator window opens with the Classic theme applied and the display
showing `0`. Use the dropdown/selector to switch between Classic, Dark (Kiro),
and Fun themes.

## 4. Running the test cases

The test suite combines property-based tests (Hypothesis) for the pure logic
with smoke, integration, and timing tests for the GUI and theming.

```bash
# Run the full test suite
pytest

# Verbose output
pytest -v
```

### Headless machines (no display server)

The GUI tests need a display. On a headless server, either skip them and run
only the pure-logic tests:

```bash
pytest -m "not gui"
```

or run everything under a virtual framebuffer:

```bash
xvfb-run -a pytest
```

### What's covered

- `tests/test_arithmetic_engine.py` — arithmetic evaluation (Property 6)
- `tests/test_digit_entry.py`, `test_decimal_handling.py` — numeric input (Properties 1, 2)
- `tests/test_operator_recording.py`, `test_operator_replacement.py` — operators (Properties 4, 5)
- `tests/test_fresh_expression.py`, `test_equals_incomplete.py`, `test_division_by_zero.py` — equals & errors (Properties 3, 7, 8)
- `tests/test_clear_reset.py`, `test_delete.py`, `test_delete_result.py`, `test_display_rendering.py` — clear/delete & display (Properties 9–12)
- `tests/test_theme_preserves_state.py`, `test_theme_manager_active.py`, `test_theme_manager_defaults.py` — theming logic (Properties 13, 14)
- `tests/test_gui_smoke.py`, `test_theme_application.py`, `test_timing.py` — GUI construction, theme application, and performance bounds
